Secure Software Development - Learning Outcomes

  • Identify and manage security risks as part of a software development project.

  • Critically analyse development problems and determine appropriate methodologies, tools and techniques (including program design and development) to solve them.

  • Design and develop/adapt computer programs and to produce a solution that meets the design brief and critically evaluate solutions that are produced.

  • Systematically develop and implement the skills required to be effective member of a development team in a virtual professional environment, adopting real-life perspectives on team roles and organisation.

    • Introduction to Secure Software Development

    Learning Outcomes

  • Identify relevant academic literature which discusses the ways in which secure software may be developed using an agile process.
  • Decompose an activity into a set of tasks which can be represented using a flow chart from the UML portfolio of modelling techniques.
  • Describe the importance of creating a culture of risk awareness in an organisation, in addition to the creation of secure software.

    • Reflection

    In the introductory week, I was drafted into Team 3 for the module projects which will be submitted in Week 6 and 11 respectively. My team mates and I created a team contract for the group and sent it to the tutor. I find the idea of working in a group quite interesting especially because my team mates have a wide range of skills and everyone communicates effectively so far.

    For the week 1 activity, I created a collaborative discussion post and flowchart on the OWASP Top 10 coding weaknesses. I have no prior knowledge of this topic however I was happy to learn about the security challenges of web applications and to increase awareness on how to create more secure software which is basically the focus of the OWASP organisation. I am really looking forward to developing skills on how to create secure solutions throughout this module.
